Tailwind CSS Animation

Add life and interactivity to your UI with Tailwind's powerful animation utilities

Summary

Animations add life to your web applications, making interactions more engaging and intuitive. Tailwind CSS provides simple utilities to implement common animations and a framework for creating custom ones. Below are the key animation types and examples for each, with brief explanations.

Note: Animation utilities in Tailwind are enabled by default. Unlike transform and transition utilities which require the appropriate classes to enable them, animation utilities work right out of the box.

Built-in Animation Utilities

Tailwind comes with several pre-defined animations that you can use immediately in your projects. These animations are perfect for common UI patterns like loading indicators, notifications, and hover effects.

Spin Animation

Perfect for loading indicators and spinners. Makes an element rotate continuously.

Spin
animate-spin 
/* CSS Definition */
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

.animate-spin {
  animation: spin 1s linear infinite;
}
View spin examples
1s duration, linear timing

Ping Animation

Useful for notifications and attention-grabbers. Creates a quick scale and fade effect.

Ping
animate-ping 
/* CSS Definition */
@keyframes ping {
  75%, 100% {
    transform: scale(2);
    opacity: 0;
  }
}

.animate-ping {
  animation: ping 1s cubic-bezier(0, 0, 0.2, 1) infinite;
}
View ping examples
1s duration, ease-out timing

Pulse Animation

Great for subtle loading states and drawing attention. Creates a gentle fading effect.

Pulse
animate-pulse 
/* CSS Definition */
@keyframes pulse {
  0%, 100% {
    opacity: 1;
  }
  50% {
    opacity: .5;
  }
}

.animate-pulse {
  animation: pulse 2s cubic-bezier(0.4, 0, 0.6, 1) infinite;
}
View pulse examples
2s duration, ease-in-out timing

Bounce Animation

Perfect for buttons and interactive elements. Creates a playful bouncing effect.

Bounce
animate-bounce 
/* CSS Definition */
@keyframes bounce {
  0%, 100% {
    transform: translateY(-25%);
    animation-timing-function: cubic-bezier(0.8, 0, 1, 1);
  }
  50% {
    transform: translateY(0);
    animation-timing-function: cubic-bezier(0, 0, 0.2, 1);
  }
}

.animate-bounce {
  animation: bounce 1s infinite;
}
View bounce examples
1s duration, special timing

Disable Animations

You can disable animations using the animate-none utility. This is useful for accessibility reasons or to stop animations conditionally.

No Animation
animate-none 
/* CSS Definition */
.animate-none {
  animation: none;
}
Accessibility considerations
Disables all animations

Custom Animation Utilities

You can define your own animations by extending the Tailwind configuration. This allows you to create animations specific to your project needs.

Extending the Configuration

To add custom animations, you need to extend the animation and keyframes sections in your Tailwind config:

// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      animation: {
        wiggle: 'wiggle 1s ease-in-out infinite',
        float: 'float 3s ease-in-out infinite',
      },
      keyframes: {
        wiggle: {
          '0%, 100%': { transform: 'rotate(-12deg)' },
          '50%': { transform: 'rotate(12deg)' },
        },
        float: {
          '0%, 100%': { transform: 'translateY(0)' },
          '50%': { transform: 'translateY(-20px)' },
        }
      }
    }
  }
}

Wiggle Animation

A playful wiggle effect that rotates an element back and forth.

Wiggle
animate-wiggle 
@keyframes wiggle {
  0%, 100% { transform: rotate(-12deg); }
  50% { transform: rotate(12deg); }
}

Float Animation

A gentle floating effect that moves an element up and down.

Float
animate-float 
@keyframes float {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-20px); }
}

Animation Properties

When defining custom animations, you can specify several properties in addition to the keyframes:

  • • Duration: How long the animation takes to complete one cycle
  • • Timing Function: The acceleration curve of the animation (linear, ease-in, etc.)
  • • Delay: Time before the animation starts
  • • Iteration Count: How many times the animation should repeat (or infinite)
  • • Direction: Whether the animation plays forward, backward, or alternates
  • • Fill Mode: How the element is styled before and after the animation

Accessibility Considerations

While animations can enhance the user experience, they can also cause problems for some users. Here are some important considerations:

Animation Can Cause Motion Sickness

Some users experience discomfort, dizziness, or nausea from animated content, especially those with vestibular disorders.

Respecting User Preferences

Users can specify that they prefer reduced motion through their operating system settings. Tailwind provides a way to respect this preference:

/* Only animate when the user hasn't specified a preference */
@media (prefers-reduced-motion: no-preference) {
  .animate-safe-spin {
    animation: spin 1s linear infinite;
  }
}

You can create variants in your Tailwind config to make this easier:

// tailwind.config.js
module.exports = {
  theme: {
    /* ... */
  },
  variants: {
    animation: ['responsive', 'motion-safe', 'motion-reduce']
  }
}

Best Practices

  • Provide controls to pause or stop animations whenever possible
  • Keep animations subtle and purposeful—avoid excessive motion
  • Respect the prefers-reduced-motion media query
  • Avoid animations that flash or pulse rapidly (can trigger seizures)
  • Use animations to provide feedback and enhance usability, not just for decoration
