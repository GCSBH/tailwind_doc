Tailwind CSS Dark Mode

Create beautiful dark themes for your website with Tailwind's intuitive dark mode utilities

Summary

Light Mode

Tailwind CSS provides a powerful dark mode feature that allows you to design both light and dark versions of your website using variant modifiers. You can create responsive, accessible dark themes with minimal effort.

Try it out: Toggle the switch above to see this page in dark mode!

Dark Mode Strategies

Tailwind CSS offers two strategies for implementing dark mode:

Class Strategy

Uses a class on the html element to control dark mode. This gives you control over when dark mode is enabled.

// tailwind.config.js
module.exports = {
  darkMode: 'class',
  // ...
}

You control dark mode with JavaScript by adding or removing the dark class to the HTML element.

Media Strategy

Uses the user's system preference to determine if dark mode should be enabled. This is controlled by the prefers-color-scheme media query.

// tailwind.config.js
module.exports = {
  darkMode: 'media',
  // ...
}

Dark mode will automatically be enabled when the user has dark mode enabled in their operating system preferences.

Using the Dark Mode Variant

To style elements differently in dark mode, use the dark: variant in front of any utility class:

<div class="bg-white dark:bg-slate-800 text-black dark:text-white">
  This div will have a white background and black text in light mode,
  but a dark slate background and white text in dark mode.
</div>

Common Dark Mode Patterns

Here are some common patterns and practices for designing effective dark mode interfaces:

Color Considerations

  • Avoid pure black (bg-black) - use dark grays like bg-slate-900 instead
  • Reduce contrast slightly - use text-gray-200 instead of text-white for body text
  • Desaturate bright colors in dark mode to be easier on the eyes
  • For dark colored elements, consider increasing their lightness in dark mode

UI Element Patterns

  • Use softer shadows or remove them entirely in dark mode
  • Consider more prominent borders (dark:border-gray-600) for separation instead of shadows
  • Adjust form elements (inputs, buttons) for better visibility in dark environments
  • Use subtle background variations (bg-gray-800 vs bg-gray-900) to create depth

Creating a Dark Mode Toggle

When using the class strategy, you'll need a way for users to toggle between light and dark mode. Here's a complete implementation:

HTML & CSS

<!-- Toggle Switch HTML -->
<div class="relative inline-block w-10 mr-2 align-middle">
  <input
    type="checkbox"
    name="toggle"
    id="toggle"
    class="toggle-checkbox absolute block w-6 h-6 
           rounded-full bg-white border-4 appearance-none 
           cursor-pointer"
  />
  <label
    for="toggle"
    class="toggle-label block overflow-hidden h-6 
           rounded-full bg-gray-300 cursor-pointer"
  ></label>
</div>
<span id="toggle-label">Light Mode</span>

<style>
  .toggle-checkbox:checked {
    right: 0;
    border-color: #4F46E5;
  }
  .toggle-checkbox:checked + .toggle-label {
    background-color: #4F46E5;
  }
</style>

JavaScript

// Toggle dark mode with JS
document.getElementById('toggle').addEventListener('change', function() {
  if (this.checked) {
    // Enable dark mode
    document.documentElement.classList.add('dark');
    document.getElementById('toggle-label').innerHTML = 'Dark Mode';
    // Save preference to localStorage
    localStorage.setItem('darkMode', 'enabled');
  } else {
    // Disable dark mode
    document.documentElement.classList.remove('dark');
    document.getElementById('toggle-label').innerHTML = 'Light Mode';
    // Save preference to localStorage
    localStorage.setItem('darkMode', 'disabled');
  }
});
// Check for saved preference (on page load)
if (localStorage.getItem('darkMode') === 'enabled') {
  document.documentElement.classList.add('dark');
  document.getElementById('toggle').checked = true;
  document.getElementById('toggle-label').innerHTML = 'Dark Mode';
}

Best Practices

Follow these best practices to create an effective dark mode experience:

Design Principles

  • Don't simply invert colors - design intentionally for dark mode
  • Ensure sufficient contrast for accessibility (WCAG standards)
  • Avoid very saturated colors which can cause eye strain in dark mode
  • Test with real users - different screens display dark colors differently

Implementation Practices

  • Remember to style all states (hover, focus, active) for dark mode
  • Save user preferences for dark/light mode in localStorage
  • Consider a transitional animation between light/dark mode
  • Use system preference as the default, but allow user overrides
